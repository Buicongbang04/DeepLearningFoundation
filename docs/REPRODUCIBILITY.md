# Reproducibility — One-Pager

Every project in this repo can be re-run end-to-end from three
commands. This page is the single source of truth for *exactly* which
commands and *exactly* which metric they should produce on a fresh
`dlcourse` env on CPU.

## Fresh-env install

```bash
git clone <repo-url>
cd Deep_Learning_Foundation

# Option A — conda.
conda env create -f environment.yml
conda activate dlcourse

# Option B — pip.
pip install -r requirements.txt
```

Then run the smoke tests to confirm the install:

```bash
pytest tests/
# expected: 46 passed (some dataset tests skipped if data is not downloaded yet)
```

## Project 01 — MLP on MNIST

```bash
python src/train.py     --config configs/mlp_mnist.yaml
python src/evaluate.py  --config configs/mlp_mnist.yaml \
                        --checkpoint experiments/mlp_mnist/checkpoints/best.pt
```

**Expected** (CPU, seed 42, 3 epochs): val_acc = **0.9700**, val_loss
≈ 0.096 at epoch 3.

## Project 03 — Transfer Learning on a Custom Dataset

```bash
# Head-only fine-tune (the headline run).
python src/train.py     --config configs/transfer_learning_custom.yaml
python src/evaluate.py  --config configs/transfer_learning_custom.yaml \
                        --checkpoint experiments/transfer_learning_custom/checkpoints/best.pt

# From-scratch baseline (for comparison).
python src/train.py     --config configs/transfer_learning_custom_scratch.yaml
python src/evaluate.py  --config configs/transfer_learning_custom_scratch.yaml \
                        --checkpoint experiments/transfer_learning_custom_scratch/checkpoints/best.pt
```

**Expected** (CPU, seed 42, 3 epochs each):
- Pretrained head-only: val_acc = **0.8000**
- From-scratch: val_acc = **0.5650**
- Headline: **gap of +23.5 percentage points** for transfer learning.

## Inference on a new input (Project 01)

```bash
python src/inference.py \
    --checkpoint experiments/mlp_mnist/checkpoints/best.pt \
    --input  <single PNG file or a directory of PNGs> \
    --output results.jsonl
```

`results.jsonl` will contain one prediction per line:

```
{"path": "image.jpg", "class": "7", "prob": 0.999}
```

## Notebooks — re-execute end-to-end

```bash
# All chapter notebooks (and labs/assignments).
for nb in notebooks/*.ipynb labs/*.ipynb assignments/*.ipynb; do
    jupyter nbconvert --to notebook --execute "$nb" --output "$(basename "$nb")"
done
```

Every notebook in this repo must finish without errors. If one
breaks, that is a bug — fix the builder script in `src/build_*.py`
and re-run.

## Notes on hardware

- All metrics above were measured on a **CPU-only** machine.
- A modern GPU will be 2–10× faster but the metric numbers should be
  bit-identical with `seed: 42` pinned on the same PyTorch version.
- Cross-PyTorch-version reproducibility is not guaranteed (a single
  decimal-place drift is normal).

## What is *not* committed

Everything in `experiments/`, `datasets/*/` (other than `README.md`
and `.gitkeep`), and `projects/*/figures/*.png` is regenerated by the
commands above. The repo's commits should never contain a checkpoint,
a downloaded dataset file, or a generated PNG.
